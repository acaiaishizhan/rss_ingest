# Feishu RSS Digest for Alibaba Cloud FC

一句话说明：这是一个部署到阿里云后，会自动定时抓取 RSS / 新闻并写入飞书的工具。

## 你在部署时主要只需要准备 6 项信息

必填：

- 飞书应用 ID
- 飞书应用 Secret
- 飞书新闻表链接
- 飞书 RSS 源表链接
- 飞书 Prompt 文档链接
- 模型 API Key

可选：

- 自动抓取频率

大多数用户只需要准备这些内容，不需要理解内部 token、table id 或其他底层配置。

## 这些信息分别是做什么的

| 你要填写的内容 | 用途 |
| :--- | :--- |
| 飞书应用 ID | 让函数能以你的飞书应用身份访问飞书 |
| 飞书应用 Secret | 和应用 ID 配合使用 |
| 飞书新闻表链接 | 把抓取结果写到这里 |
| 飞书 RSS 源表链接 | 从这里读取要抓取的 RSS 源 |
| 飞书 Prompt 文档链接 | 从这里读取模型分析提示词 |
| 模型 API Key | 用来做摘要、分类和评分 |
| 自动抓取频率（可选） | 控制它多久自动跑一次，默认每小时一次 |

## 部署体验会是什么样

- 你在阿里云部署页面里，主要就是填写上面这些信息。
- 部署成功后，函数会自动按定时任务运行。
- 平时不需要你每次手动进去点击运行。
- 你真正长期维护的，通常只有：
  - RSS 源表里的订阅源
  - Prompt 文档里的分析要求

## 自动运行说明

- 这个项目默认开启自动定时抓取。
- 默认频率是每小时一次。
- 如果你想改频率，只需要改一个入口：自动抓取频率。
- 不需要同时理解多套频率参数。

## 哪些问题会影响运行结果

下面这些问题会影响抓取结果，但不等于“阿里云部署失败”：

- RSS 源本身无法访问，或者在当前网络环境下经常超时。
- 飞书应用权限没有配好，导致表格读写失败，或者文档读取失败。
- 模型 API 无法访问，尤其是当前版本默认依赖海外模型接口。
- 飞书链接填错，导致函数虽然启动了，但读写位置不对。

如果出现这些情况，通常需要查看函数运行日志继续排查。

## 适合谁用

- 想把 RSS 抓取长期托管到阿里云的人。
- 想把新闻自动沉淀到飞书表格里的人。
- 想让系统自动定时运行，而不是自己手动跑脚本的人。

## 部署前提醒

- 这不是“点一下就一定无脑成功”的模板。
- 它更像是一套已经接好的自动化定时任务。
- 部署成功后会自动跑，但前提是飞书、RSS 和模型接口本身要可用。

## 高级兼容说明

- 当前版本内部仍兼容旧版底层定位方式和旧版频率字段，主要用于兼容老部署。
- 新部署建议只使用正文里的 6 项必填信息，以及 1 项可选频率。
- 如果你不是在做历史迁移，可以忽略这些底层兼容项。
