# Feishu RSS Digest for Alibaba Cloud FC

一句话说明：这是一个部署到阿里云后，会自动定时抓取 RSS / 新闻并写入飞书的工具。

## 你只需要改一个文件

部署时，用户只需要修改仓库根目录的 `填写这里.yml`。

你不需要再去阿里云控制台补环境变量。
底层会自动把这个文件里的业务参数映射成函数运行所需的配置。

## 部署步骤

1. fork 这个仓库到你自己的账号
2. 打开仓库根目录的 `填写这里.yml`
3. 按注释填入你的业务参数
4. 部署到阿里云

## 填写这里.yml 里要填什么

必填：

- 飞书应用 ID
- 飞书应用 Secret
- 飞书新闻表链接
- 飞书 RSS 源表链接
- 飞书 Prompt 文档链接
- 模型 API Key

可选：

- 自动抓取间隔分钟数

默认是 `60`，也就是每小时一次。

## 这些字段分别是做什么的

| 你要填写的内容 | 用途 |
| :--- | :--- |
| 飞书应用 ID | 让函数能以你的飞书应用身份访问飞书 |
| 飞书应用 Secret | 和应用 ID 配合使用 |
| 飞书新闻表链接 | 把抓取结果写到这里 |
| 飞书 RSS 源表链接 | 从这里读取要抓取的 RSS 源 |
| 飞书 Prompt 文档链接 | 从这里读取模型分析提示词 |
| 模型 API Key | 用来做摘要、分类和评分 |
| 自动抓取间隔分钟数（可选） | 填一个数字，表示多少分钟抓取一次，默认 60 |

## 部署后会怎样运行

- 部署成功后，函数会自动按定时任务运行。
- 默认频率是每小时一次。
- 如果你想改频率，只需要改 `填写这里.yml` 里的分钟数。
- 平时不需要你每次手动进入控制台点击运行。

## 哪些问题会影响运行结果

下面这些问题会影响抓取结果，但不等于“阿里云部署失败”：

- RSS 源本身无法访问，或者在当前网络环境下经常超时。
- 飞书应用权限没有配好，导致表格读写失败，或者文档读取失败。
- 模型 API 无法访问，尤其是当前版本默认依赖海外模型接口。
- 飞书链接填错，导致函数虽然启动了，但读写位置不对。

如果出现这些情况，通常需要查看函数运行日志继续排查。

## 适合谁用

- 想把 RSS 抓取长期托管到阿里云的人。
- 想把新闻自动沉淀到飞书表格里的人。
- 想让系统自动定时运行，而不是自己手动跑脚本的人。

## 高级说明

- 本项目的业务代码仍然使用现有环境变量名，但这些变量会由 `填写这里.yml` 自动映射生成。
- `填写这里.yml` 里的分钟数会在 `s.yaml` 中自动转换成阿里云函数计算需要的定时表达式。
- 如果你不是在做底层兼容或二次开发，可以不用关心这些实现细节。
